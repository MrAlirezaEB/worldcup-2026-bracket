# BUGLOOS WorldCup 2026 Bracket

این پروژه یک اپلیکیشن پیش‌بینی مرحله گروهی جام جهانی ۲۰۲۶ است که با استفاده از Node.js (بک‌ند) و Vue.js (فرانت‌ند) ساخته شده است.

## ویژگی‌ها
- ورود با ایمیل (هر ایمیل فقط یک بار مجاز به ثبت است)
- قابلیت Drag & Drop برای تعیین رتبه‌بندی تیم‌ها در ۱۲ گروه (A تا L)
- ذخیره‌سازی در فایل JSON (مناسب برای نسخه‌های سبک و تست)
- رابط کاربری مدرن و جذاب با Tailwind CSS v4

## پیش‌نیازها
- Docker و Docker Compose (پیشنهادی برای اجرا)
- Node.js (v18 یا بالاتر - برای اجرای محلی بدون داکر)

## استقرار (Deployment) با Docker

بهترین راه برای اجرای پروژه در محیط تولید استفاده از Docker است.

### استفاده از Makefile (پیشنهادی)

برای راحتی بیشتر، دستورات رایج در یک `Makefile` آماده شده‌اند:

```bash
# بیلد کردن تصاویر (Images)
make build

# اجرا کردن کانتینرها در پس‌زمینه
make up

# مشاهده لاگ‌ها
make logs

# متوقف کردن کانتینرها
make down
```

### استفاده مستقیم از Docker Compose

اگر `make` روی سیستم شما نصب نیست، می‌توانید مستقیماً از دستورات داکر استفاده کنید:

```bash
# بیلد و اجرا
docker-compose up --build -d
```

پس از اجرا، اپلیکیشن در آدرس `http://localhost:7021` در دسترس خواهد بود.

### تنظیم پورت و دسترسی
اگر پورت ۷۰۲۱ سرور شما اشغال شده است، می‌توانید پورت دیگری را در فایل `.env` یا مستقیماً در دستور اجرا تنظیم کنید:
```bash
APP_PORT=9090 make up
```

### تنظیم دامین (اختیاری)
اگر می‌خواهید از دامین استفاده کنید و **Nginx ندارید**، بهترین راه‌ها این موارد هستند:

#### روش ۱: استفاده از Cloudflare Origin Rules (سریع‌ترین)
اگر دامنه شما روی Cloudflare است:
1. در پنل Cloudflare به بخش **Rules > Origin Rules** بروید.
2. یک Rule جدید بسازید.
3. در بخش **If incoming requests match**، فیلد `Hostname` را انتخاب کرده و دامین خود را وارد کنید.
4. در بخش **Destination Port**، گزینه `Rewrite to` را انتخاب کرده و پورت **7021** را وارد کنید.
5. حالا ترافیک دامین شما مستقیماً به پورت ۷۰۲۱ سرور می‌رسد و SSL هم توسط Cloudflare مدیریت می‌شود.

#### روش ۲: استفاده از Cloudflare Tunnel (امن‌ترین و بهترین)
در این روش نیازی به باز کردن هیچ پورتی روی فایروال سرور نیست:
1. در کنسول [Cloudflare Zero Trust](https://one.dash.cloudflare.com/) یک Tunnel بسازید.
2. توکن (Token) ارائه شده را کپی کنید.
3. در فایل `docker-compose.yml` بخش مربوط به `tunnel` را از حالت کامنت خارج کنید.
4. دستور اجرا را به این شکل بزنید:
   ```bash
   TUNNEL_TOKEN=your_token_here make up
   ```

#### روش ۳: استفاده از Nginx (اگر نصب است)
اگر Nginx روی سرور دارید، یک کانفیگ مشابه زیر اضافه کنید:
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:7021;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

اگر از **Cloudflare** استفاده می‌کنید، می‌توانید در بخش "Origin Rules"، ترافیک دامنه را مستقیماً به پورت 7021 سرور خود هدایت کنید.

برای تغییر پورت پیش‌فرض (7021) به پورت دیگر، می‌توانید از متغیر محیطی در فایل `.env` یا مستقیماً در دستور اجرا استفاده کنید:
```bash
APP_PORT=9090 make up
```

## نحوه اجرا (بدون Docker)

### ۱. تنظیمات
به صورت پیش‌فرض برنامه در پورت 5000 اجرا می‌شود. می‌توانید این مقدار را در فایل `backend/.env` تغییر دهید:
```env
PORT=5000
```
داده‌ها در فایل `backend/db.json` ذخیره می‌شوند.

### ۲. نصب و اجرا
در پوشه اصلی پروژه دستورات زیر را اجرا کنید:

```bash
# رفتن به پوشه بک‌ند
cd backend

# نصب وابستگی‌ها
npm install

# اجرای سرور
node server.js
```

حالا اپلیکیشن در آدرس `http://localhost:5000` در دسترس خواهد بود.

## ساختار پروژه
- `backend/`: شامل کد سرور اکسپرس (Node.js) و ذخیره‌سازی داده‌ها در فایل `db.json`
- `frontend/`: شامل کد Vue 3 و استایل‌های Tailwind. در محیط داکر، از Caddy برای سرو فایل‌ها استفاده می‌شود.
